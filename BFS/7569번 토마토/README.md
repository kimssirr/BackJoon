# 토마토

### 요구사항
- 3차원(가로 n × 세로 m × 높이 h)의 토마토 상자가 주어진다.
- 1: 익은 토마토
- 0: 익지 않은 토마토
- -1: 토마토가 없는 칸
- 하루가 지나면 익은 토마토(1) 는 상하좌우앞뒤 6방향의 익지 않은 토마토(0)를 익게 만든다.
- 모든 토마토가 익는 데 걸리는 최소 날짜를 출력해야 한다.
- 단, 익을 수 없는 토마토가 존재하면 -1 을 출력한다.
- 이미 모든 토마토가 익어 있는 경우 0 출력.

### 풀이
1. 3차원 배열 map[h][m][n] 에 토마토 상태를 입력받는다.
2. 입력받는 과정에서
    - 1(익은 토마토)이면 BFS 시작점이므로 queue에 push
    - 0(안 익은 토마토) 개수를 zero 변수로 세어둔다
3. 안 익은 토마토가 **zero == 0** 이면, 처음부터 다 익어 있으므로 0 출력 후 종료.
4. BFS 실행
    - 기본적으로 익은 토마토(1)를 시작점으로
    - queue에서 하나씩 꺼내면서 6방향(nx, ny, nz) 탐색
    - 유효성 검사
        - 범위 안쪽
        - -1이 아님
        - 아직 익지 않은 칸(map == 0)
    - 조건을 만족하면
        - 그 칸의 날짜 = 현재 칸 날짜 + 1
        - queue에 push
5. BFS 종료 후 전체 map을 다시 순회
    - 여전히 map[z][y][x] == 0 이 남아 있다면 익힐 수 없으므로 -1 출력
    - 그렇지 않다면 map 값들 중 최대값을 max로 저장
6. 최종 정답은 **max - 1**
    - 처음 익어 있던 토마토들이 1이므로, 익는 데 걸린 날짜는 (최대값 - 1)

### 시간복잡도
- 3차원 배열 전체 탐색 + BFS
- 모든 칸을 최대 한 번씩 방문 → O(n × m × h)
