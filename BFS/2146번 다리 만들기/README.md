# 다리 만들기

### 요구사항
- N × N 격자에서 여러 개의 섬이 존재한다.
- 서로 다른 두 섬을 연결하는 **가장 짧은 다리 길이**를 구한다.
- 다리는 바다(0)만 지날 수 있으며, 상하좌우로만 확장 가능하다.

### 풀이

#### 1. 섬 구분 및 가장자리 수집 (`mapRegen`)

1. 전체 격자를 2중 for문으로 순회한다.
2. 방문하지 않은 육지(1)를 만나면 BFS를 실행해
    - 같은 섬을 같은 숫자(a)로 `map`을 재구성한다.
3. BFS 중, 현재 칸의 주변에 바다(0)가 하나라도 있으면
    - 해당 칸을 **섬의 가장자리**로 판단하고 `startQ`에 저장한다.
4. 하나의 섬 처리가 끝날 때마다 섬 번호를 증가시킨다.

→ 이 단계가 끝나면
- 각 섬은 서로 다른 숫자로 구분되어 있고
- 모든 섬의 가장자리 좌표가 `startQ`에 들어 있다.

---

#### 2. 다리 확장 BFS (`BFS`)

1. `startQ`를 시작 큐로 사용해 BFS를 진행한다.
2. 각 가장자리에서 바다(0)로 확장하며
    - 바다는 자신의 섬 번호로 바꾸고
    - `dist[][]`로 거리(다리 길이)를 관리한다.
3. BFS 중,
    - **자기 섬이 아닌 다른 섬 번호를 만나면**
    - `현재 위치 거리 + 상대 위치 거리`로 다리 길이를 계산한다.
4. 그중 최솟값을 갱신하며 BFS를 끝까지 수행한다.

---

### 결과
- 서로 다른 두 섬이 처음 만나는 순간의 거리 합이
  **최단 다리 길이**가 된다.

### 시간복잡도
- 섬 구분 BFS: **O(N × N)**
- 다리 확장 BFS: **O(N × N)**
- 전체 시간복잡도: **O(N × N)**
- 공간복잡도: **O(N × N)**
